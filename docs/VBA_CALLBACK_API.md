# VBA Callback API Contract

This document describes the API contract between the MCP server and the VBA add-in for async operations with progress reporting.

## Overview

The MCP server supports two VBA entry points:

| Entry Point | Signature | Use Case |
|-------------|-----------|----------|
| `API` | `API(strCommand, [args...])` | Sync calls - returns result immediately |
| `APIAsync` | `APIAsync(strCallbackInfo, strCommand, [args...])` | Async calls - spawns detached process, uses callbacks |

## Callback Info Parameter

When calling `APIAsync`, the MCP server passes a JSON string as the first parameter:

```json
{
    "callback_url": "http://127.0.0.1:54321/callback",
    "operation_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
    "client": "cursor"
}
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `callback_url` | Yes | Full URL to POST callbacks to |
| `operation_id` | Yes | UUID generated by MCP server (include in every callback) |
| `client` | Optional | Calling tool identifier (cursor, claude-code, vscode) |

## VBA Implementation

### Existing Sync API (Unchanged)

```vba
' ============================================================
' EXISTING SYNC API (unchanged, backward compatible)
' ============================================================
Public Function API(strCommand As String, Optional varArg1 As Variant, _
                    Optional varArg2 As Variant) As Variant
    Select Case strCommand
        Case "GetVCSVersion": API = VCS_VERSION
        Case "GetOptions": API = GetOptionsJson()
        Case "Export": API = VCS.Export(varArg1)  ' Blocks until complete
        Case "Build": API = VCS.Build(varArg1)    ' Blocks until complete
        ' ... existing behavior unchanged
    End Select
End Function
```

### New Async API

```vba
' ============================================================
' NEW ASYNC API (for MCP with callbacks)
' ============================================================
Public Function APIAsync(strCallbackInfo As String, strCommand As String, _
                         Optional varArg1 As Variant, Optional varArg2 As Variant) As String
    
    Dim callback As Dictionary
    Dim callbackUrl As String, operationId As String
    
    ' Parse callback info
    Set callback = JsonConverter.ParseJson(strCallbackInfo)
    callbackUrl = callback("callback_url")
    operationId = callback("operation_id")
    
    Select Case strCommand
        Case "Export"
            ' Spawn detached process via Timer API
            StartAsyncExport callbackUrl, operationId, varArg1
            ' Return immediately with timeout hint
            APIAsync = "{""async"":true,""timeout_ms"":300000}"
            
        Case "ExportVBA"
            StartAsyncExportVBA callbackUrl, operationId, varArg1
            APIAsync = "{""async"":true,""timeout_ms"":120000}"
            
        Case "Build"
            StartAsyncBuild callbackUrl, operationId, varArg1
            APIAsync = "{""async"":true,""timeout_ms"":600000}"
            
        Case "BuildAs"
            StartAsyncBuildAs callbackUrl, operationId, varArg1
            APIAsync = "{""async"":true,""timeout_ms"":600000}"
            
        Case "MergeBuild"
            StartAsyncMergeBuild callbackUrl, operationId
            APIAsync = "{""async"":true,""timeout_ms"":300000}"
            
        Case Else
            ' Fall back to sync for commands that don't need async
            Dim syncResult As Variant
            syncResult = API(strCommand, varArg1, varArg2)
            APIAsync = "{""sync"":true,""result"":" & JsonConverter.ConvertToJson(syncResult) & "}"
    End Select
End Function
```

## APIAsync Return Values

### Sync Response

For quick commands routed through APIAsync that don't need async execution:

```json
{"sync": true, "result": "4.1.4"}
```

### Async Response

For operations that spawn a detached process:

```json
{"async": true, "timeout_ms": 300000}
```

The `timeout_ms` field tells the MCP server how long to wait for completion callbacks.

## Sending Callbacks

VBA sends callbacks to the MCP server via HTTP POST:

### Utility Function

```vba
' Utility function for posting callbacks
Public Sub PostCallback(callbackUrl As String, operationId As String, _
                        msgType As String, Optional progress As Long = -1, _
                        Optional total As Long = -1, Optional message As String = "")
    
    ' Skip if no callback URL
    If Len(callbackUrl) = 0 Then Exit Sub
    
    On Error Resume Next  ' Don't let callback failures break the operation
    
    Dim http As Object
    Dim json As String
    
    Set http = CreateObject("MSXML2.XMLHTTP")
    
    ' Build JSON payload
    json = "{" & _
           """operation_id"":""" & operationId & """," & _
           """type"":""" & msgType & """"
    
    If progress >= 0 Then json = json & ",""progress"":" & progress
    If total >= 0 Then json = json & ",""total"":" & total
    If Len(message) > 0 Then json = json & ",""message"":""" & EscapeJson(message) & """"
    
    json = json & "}"
    
    http.Open "POST", callbackUrl, False
    http.setRequestHeader "Content-Type", "application/json"
    http.send json
    
    On Error GoTo 0
End Sub

' Helper to escape JSON strings
Private Function EscapeJson(str As String) As String
    EscapeJson = Replace(str, "\", "\\")
    EscapeJson = Replace(EscapeJson, """", "\""")
    EscapeJson = Replace(EscapeJson, vbCr, "\r")
    EscapeJson = Replace(EscapeJson, vbLf, "\n")
    EscapeJson = Replace(EscapeJson, vbTab, "\t")
End Function
```

### Callback Endpoint

**URL:** `POST http://127.0.0.1:{port}/callback`

**Request Body (JSON):**

```json
{
    "operation_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
    "type": "progress",
    "progress": 50,
    "total": 100,
    "message": "Exporting queries..."
}
```

**Response:** `200 OK` with `{"received": true}`

## Callback Message Types

| Type | Purpose | Required Fields | When to Send |
|------|---------|-----------------|--------------|
| `progress` | Progress update | `progress` | Periodically during operation |
| `log` | Informational message | `message`, `level` | Notable events |
| `error` | Operation failed | `message` | When operation encounters an error (ends wait) |
| `complete` | Operation succeeded | (none) | At end of successful operation (ends wait) |
| `cancelled` | Operation was cancelled | (none) | When operation aborts due to cancellation (ends wait) |

### Progress Callback

```json
{
    "operation_id": "uuid-here",
    "type": "progress",
    "progress": 50,
    "total": 100,
    "message": "Exporting queries (15 of 30)..."
}
```

### Log Callback

```json
{
    "operation_id": "uuid-here",
    "type": "log",
    "level": "info",
    "message": "Skipping unchanged object: qryCustomers"
}
```

Supported levels: `debug`, `info`, `warning`, `error`

### Complete Callback

```json
{
    "operation_id": "uuid-here",
    "type": "complete",
    "progress": 100,
    "total": 100,
    "message": "Export completed successfully"
}
```

### Error Callback

```json
{
    "operation_id": "uuid-here",
    "type": "error",
    "message": "Export failed: File locked by another process",
    "code": "FILE_LOCKED"
}
```

### Cancelled Callback

```json
{
    "operation_id": "uuid-here",
    "type": "cancelled",
    "message": "Operation cancelled by user"
}
```

## Cancellation Support

Operations can be cancelled by either:
1. **MCP side** - Agent calls `access_cancel_operation(operation_id)`
2. **VBA side** - User cancels via UI, VBA sends `cancelled` callback

### Cancellation Flow

```
MCP Server                              VBA Add-in
    |                                       |
    |-- request_cancel(operation_id) ------>|
    |                                       |
    |   VBA polls during DoEvents:          |
    |<-- GET /cancel-status/{operation_id} -|
    |-- {"cancelled": true} --------------->|
    |                                       |
    |   VBA aborts operation:               |
    |<-- POST /callback {type:"cancelled"} -|
    |                                       |
```

### Check Cancellation (VBA)

VBA should poll for cancellation during `DoEvents` calls. Use throttling to avoid excessive HTTP requests:

```vba
' Module-level variables for throttling
Private m_LastCancelCheck As Double
Private Const CANCEL_CHECK_INTERVAL_MS As Long = 500  ' Check every 500ms

' Check if operation should be cancelled (call during DoEvents)
Public Function CheckCancelled(callbackUrl As String, operationId As String) As Boolean
    On Error Resume Next
    
    ' Throttle checks to avoid excessive HTTP requests
    Dim now As Double
    now = Timer * 1000  ' Convert to milliseconds
    
    If now - m_LastCancelCheck < CANCEL_CHECK_INTERVAL_MS Then
        CheckCancelled = False
        Exit Function
    End If
    m_LastCancelCheck = now
    
    ' Build cancel-status URL from callback URL
    ' callback_url: http://127.0.0.1:54321/callback
    ' cancel_url:   http://127.0.0.1:54321/cancel-status/{operation_id}
    Dim cancelUrl As String
    cancelUrl = Replace(callbackUrl, "/callback", "/cancel-status/" & operationId)
    
    ' Check cancel status
    Dim http As Object
    Set http = CreateObject("MSXML2.XMLHTTP")
    http.Open "GET", cancelUrl, False
    http.send
    
    If http.Status = 200 Then
        Dim response As Dictionary
        Set response = JsonConverter.ParseJson(http.responseText)
        CheckCancelled = response("cancelled")
    Else
        CheckCancelled = False
    End If
    
    On Error GoTo 0
End Function

' Handle cancellation in operation loop
Private Sub RunAsyncExport(callbackUrl As String, operationId As String, _
                           exportPath As String)
    On Error GoTo ErrorHandler
    
    Dim i As Long, total As Long
    
    total = CurrentDb.QueryDefs.Count
    For i = 0 To total - 1
        ' Check for cancellation during DoEvents
        DoEvents
        If CheckCancelled(callbackUrl, operationId) Then
            PostCallback callbackUrl, operationId, "cancelled", -1, -1, _
                         "Export cancelled by user"
            Exit Sub
        End If
        
        ExportQuery CurrentDb.QueryDefs(i), exportPath
        PostCallback callbackUrl, operationId, "progress", _
                     CLng((i + 1) * 100 / total), 100, _
                     "Exporting query: " & CurrentDb.QueryDefs(i).Name
    Next i
    
    PostCallback callbackUrl, operationId, "complete", 100, 100, "Export finished"
    Exit Sub
    
ErrorHandler:
    PostCallback callbackUrl, operationId, "error", -1, -1, _
                 "Export failed: " & Err.Description
End Sub
```

### Cancel from VBA Side

If the user cancels via a VBA UI (button, Escape key, etc.):

```vba
' User clicked Cancel button
Private Sub btnCancel_Click()
    ' Set local flag (checked in operation loop)
    m_CancelRequested = True
    
    ' Optionally notify MCP immediately
    PostCallback m_CallbackUrl, m_OperationId, "cancelled", -1, -1, _
                 "Cancelled by user"
End Sub
```

### HTTP Endpoints for Cancellation

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/cancel-status/{operation_id}` | GET | Check if operation should be cancelled |
| `/cancel/{operation_id}` | POST | Request cancellation (from VBA or MCP) |

**GET /cancel-status/{operation_id}**

```json
{"operation_id": "uuid-here", "cancelled": true}
```

**POST /cancel/{operation_id}**

```json
{"operation_id": "uuid-here", "cancelled": true, "message": "Cancellation requested"}
```

## Example: Async Export with Cancellation

```vba
' Called via Timer API to run detached from COM call
Private Sub RunAsyncExport(callbackUrl As String, operationId As String, _
                           exportPath As String)
    On Error GoTo ErrorHandler
    
    Dim i As Long, total As Long
    
    ' Report start
    PostCallback callbackUrl, operationId, "progress", 0, 100, "Starting export..."
    
    ' Export queries (0-30%)
    total = CurrentDb.QueryDefs.Count
    For i = 0 To total - 1
        ' Check for cancellation
        DoEvents
        If CheckCancelled(callbackUrl, operationId) Then GoTo Cancelled
        
        ExportQuery CurrentDb.QueryDefs(i), exportPath
        PostCallback callbackUrl, operationId, "progress", _
                     CLng((i + 1) * 30 / total), 100, _
                     "Exporting query: " & CurrentDb.QueryDefs(i).Name
    Next i
    
    ' Export modules (30-60%)
    DoEvents
    If CheckCancelled(callbackUrl, operationId) Then GoTo Cancelled
    PostCallback callbackUrl, operationId, "progress", 30, 100, "Exporting modules..."
    ' ... export modules with similar cancel checks ...
    
    ' Export forms (60-80%)
    DoEvents
    If CheckCancelled(callbackUrl, operationId) Then GoTo Cancelled
    PostCallback callbackUrl, operationId, "progress", 60, 100, "Exporting forms..."
    ' ... export forms ...
    
    ' Export reports (80-100%)
    DoEvents
    If CheckCancelled(callbackUrl, operationId) Then GoTo Cancelled
    PostCallback callbackUrl, operationId, "progress", 80, 100, "Exporting reports..."
    ' ... export reports ...
    
    ' Report completion
    PostCallback callbackUrl, operationId, "complete", 100, 100, "Export finished"
    Exit Sub
    
Cancelled:
    PostCallback callbackUrl, operationId, "cancelled", -1, -1, "Export cancelled"
    Exit Sub
    
ErrorHandler:
    PostCallback callbackUrl, operationId, "error", -1, -1, _
                 "Export failed: " & Err.Description
End Sub
```

## Concurrent Operation Handling

The MCP server tracks active operations per database to prevent conflicts. Since Access/VBA is single-threaded and VCS operations are designed to run one at a time, concurrent requests to the same database receive an immediate "busy" response.

### Busy Response

When an operation is already running for a database:

```json
{
    "success": false,
    "error": "Database is busy: Export in progress",
    "busy": true,
    "active_operation_id": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
    "active_command": "Export",
    "elapsed_seconds": 45.2,
    "hint": "Wait for the current operation to complete, or cancel it with access_cancel_operation()"
}
```

### Agent Behavior

When an agent receives a busy response, it can:

1. **Wait** - Retry after the current operation completes
2. **Cancel** - Call `access_cancel_operation(active_operation_id)` if the user wants to interrupt
3. **Report** - Inform the user that an operation is in progress

### VBA Considerations

The MCP server handles concurrency tracking. VBA doesn't need to implement additional locking, but should:

- Complete operations cleanly (send `complete` or `error` callback)
- Check for cancellation regularly during long operations
- Not assume an operation_id is still valid after completion

## Cross-Tool Compatibility

This API works with any MCP-compatible tool:

| Tool | Client ID | Notes |
|------|-----------|-------|
| Cursor | `cursor` | Primary development target |
| Claude Code | `claude-code` | Anthropic's CLI tool |
| VS Code + Continue | `vscode-continue` | Open source extension |
| Custom MCP clients | `custom` | Any tool implementing MCP |

The `client` field is informational - VBA behavior should be consistent regardless of calling tool.

## Future Extensibility

The JSON callback parameter allows adding fields without breaking existing integrations:

```json
{
    "callback_url": "http://127.0.0.1:54321/callback",
    "operation_id": "uuid-here",
    "client": "cursor",
    "cancel_url": "http://127.0.0.1:54321/cancel/uuid-here",
    "timeout_ms": 300000,
    "log_level": "debug"
}
```

VBA code should ignore unknown fields for forward compatibility.
